# CramCraft — Smart Flashcard Engine

> *"The craft of smart studying"* — AI-powered flashcard engine that turns any PDF or PPTX into smart, practice-ready decks with spaced repetition, revision sheets, and exam prep.

**Live Demo → https://cram-craft.vercel.app/**

---

## Overview

CramCraft is a full-stack study tool where you drop in a PDF or PowerPoint and get back a rich set of flashcards, revision sheets, and exam questions — all generated by AI. Cards are scheduled using the SM-2 spaced repetition algorithm so you review each card at exactly the right time.

Every user's data is isolated behind Google OAuth — sign in and your decks, stats, and progress follow you across devices. AI generation is powered by **Groq** (free tier) using `llama-3.3-70b-versatile`, and data is persisted in **Supabase** (PostgreSQL).

---

## Features

- **PDF & PPTX upload** — drag-and-drop, up to 500 MB, server-side text extraction
- **AI flashcard generation** — Groq LLM generates rich cards with answers, explanations, mnemonics, key formulas, and exam tips
- **Revision sheets** — visual study aids grouped by topic, generated alongside flashcards
- **SM-2 spaced repetition** — cards are scheduled based on your self-ratings (Again / Hard / Good / Easy)
- **Exam prep mode** — curated Q&A ranked by importance with timed sessions and scoring
- **Activity heatmap** — GitHub-style study streak tracker on the Stats page
- **Mastery tracking** — per-deck mastery percentage with circular progress rings
- **Print / Save as PDF** — dedicated print-optimised page for any deck
- **Backup export** — download any deck as a formatted `.txt` study document
- **Dark mode** — persisted via Zustand + localStorage
- **Google OAuth** — per-user data isolation, every deck and session scoped to the signed-in user
- **Keyboard shortcuts** — Space to flip, 1–4 to rate during study sessions

## Database Schema

Run the following SQL in the **Supabase SQL Editor** to initialise your schema:

```sql
-- Decks (scoped per user)
CREATE TABLE decks (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL DEFAULT 'local',
  title TEXT NOT NULL,
  description TEXT,
  source_filename TEXT,
  card_count INTEGER DEFAULT 0,
  mastery_percentage REAL DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  last_studied_at TIMESTAMPTZ
);

-- Flashcards
CREATE TABLE cards (
  id TEXT PRIMARY KEY,
  deck_id TEXT NOT NULL REFERENCES decks(id) ON DELETE CASCADE,
  front TEXT NOT NULL,
  back TEXT NOT NULL,         -- JSON: { answer, explanation, mnemonic, keyFormula, examTip }
  card_type TEXT DEFAULT 'concept',
  repetitions INTEGER DEFAULT 0,
  easiness_factor REAL DEFAULT 2.5,
  interval_days INTEGER DEFAULT 0,
  next_review_at TIMESTAMPTZ DEFAULT NOW(),
  last_reviewed_at TIMESTAMPTZ
);

-- Study activity (heatmap / streaks)
CREATE TABLE study_stats (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL DEFAULT 'local',
  date TEXT NOT NULL,
  cards_studied INTEGER DEFAULT 0,
  UNIQUE(user_id, date)
);

-- Exam sessions
CREATE TABLE exam_sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL DEFAULT 'local',
  deck_id TEXT,
  title TEXT NOT NULL,
  questions TEXT NOT NULL,   -- JSON array
  answers TEXT,
  score REAL,
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

> **Row Level Security:** Enable RLS on all tables in Supabase and add `user_id`-scoped policies for an extra layer of data isolation beyond the application-level checks.

## Environment Variables

Copy `.env.local.example` to `.env.local` and fill in your values:

```env
# ── AI: Groq (free) ──────────────────────────────────────────
GROQ_API_KEY=gsk_...

# ── Auth: NextAuth ────────────────────────────────────────────
NEXTAUTH_SECRET=your-random-secret-here
NEXTAUTH_URL=http://localhost:3006

# ── Auth: Google OAuth ────────────────────────────────────────
GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret

# ── Database: Supabase ────────────────────────────────────────
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

### Getting credentials

**Groq API key (free):**
1. Sign up at [console.groq.com](https://console.groq.com) — no credit card required
2. API Keys → Create API Key
3. Copy the key starting with `gsk_`

**Google OAuth:**
1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. APIs & Services → Credentials → Create OAuth 2.0 Client ID
3. Authorised redirect URIs:
   - Dev: `http://localhost:3006/api/auth/callback/google`
   - Prod: `https://yourdomain.com/api/auth/callback/google`

**NEXTAUTH_SECRET:**
```bash
openssl rand -base64 32
```

**Supabase:**
1. Create a project at [supabase.com](https://supabase.com)
2. Go to **Settings → API**
3. Copy **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
4. Copy **service_role** secret → `SUPABASE_SERVICE_ROLE_KEY`
5. Run the schema SQL above in the **SQL Editor**

## API Routes

| Method | Route | Description |
|---|---|---|
| `POST` | `/api/generate` | PDF/PPTX text → Groq → flashcards + revision sheets |
| `GET` | `/api/decks` | List all decks for the signed-in user |
| `POST` | `/api/decks` | Create a new deck with cards |
| `GET` | `/api/decks/[id]` | Get deck with all cards and stats |
| `DELETE` | `/api/decks/[id]` | Delete deck and all its cards |
| `GET` | `/api/decks/[id]/export` | Download deck as formatted `.txt` |
| `GET` | `/api/study/[deckId]` | Cards due for review (SM-2 ordered) |
| `POST` | `/api/cards/[id]/review` | Submit SM-2 rating, reschedule card |
| `GET` | `/api/exam` | List exam sessions |
| `POST` | `/api/exam` | Generate a new exam from a deck |
| `GET` | `/api/exam/[id]` | Get exam with questions |
| `PATCH` | `/api/exam/[id]` | Submit exam answers, calculate score |
| `GET` | `/api/stats` | Study streak, daily history, heatmap data |

All routes return `401` if not authenticated, `403` if the resource belongs to another user.

## Spaced Repetition — SM-2

Self-rating scale used during study sessions:

| Button | Quality | Meaning |
|---|---|---|
| Again | 0 | Completely forgot |
| Hard | 2 | Recalled with major difficulty |
| Good | 3 | Recalled after hesitation |
| Easy | 5 | Perfect recall |

A card is **mastered** when `repetitions >= 3` AND `easiness_factor >= 2.0`. New cards start at 0% mastery — this is expected. Study consistently and mastery climbs automatically.

## Deployment

### Vercel + Supabase (recommended)

Supabase provides a persistent PostgreSQL database that works seamlessly with Vercel's serverless functions — no ephemeral storage or cold-start database concerns.

```bash
# Push to GitHub, then import the repo at vercel.com
# Add all .env.local variables under Settings → Environment Variables
# Update NEXTAUTH_URL to your production URL
# Update Google OAuth redirect URI to https://yourdomain.com/api/auth/callback/google
```

**Supabase production checklist:**
- Enable **Row Level Security (RLS)** on all tables
- Add `user_id`-scoped policies to prevent cross-user data access
- Use the `service_role` key only in server-side routes — never expose it client-side
- `NEXT_PUBLIC_SUPABASE_URL` is safe to expose; `SUPABASE_SERVICE_ROLE_KEY` is not

## Security

- API keys live only in `.env.local` — never in client code or `NEXT_PUBLIC_` variables
- `SUPABASE_SERVICE_ROLE_KEY` is used exclusively in server-side API routes
- `GROQ_API_KEY` is server-side only — never exposed to the client
- All `/api/*` routes call `getAuthUserId()` and return `401` if the session is missing
- Deck/exam ownership is verified on every write operation (`403` if user_id mismatch)
- `middleware.ts` protects all app routes — unauthenticated users are redirected to `/auth/signin`
- Google OAuth uses `prompt: 'select_account'` to always show the account picker

# CramCraft ‚Äî Smart Flashcard Engine

> Turn any PDF or PPTX into a smart, practice-ready deck of flashcards with AI and spaced repetition.
> 
LIVE DEMO-- https://cram-craft.vercel.app/
---

## Overview

CramCraft is a full-stack study tool where you drop in a PDF or PowerPoint and get back a rich set of flashcards, revision sheets, and exam questions ‚Äî all generated by Claude AI. Cards are scheduled using the SM-2 spaced repetition algorithm so you review each card at exactly the right time.

Every user's data is isolated behind Google OAuth ‚Äî sign in and your decks, stats, and progress follow you across devices.

---

## Features

- **PDF & PPTX upload** ‚Äî drag-and-drop, up to 500 MB, server-side text extraction
- **AI flashcard generation** ‚Äî Claude generates rich cards with answers, explanations, mnemonics, key formulas, and exam tips
- **Revision sheets** ‚Äî visual study aids grouped by topic, generated alongside flashcards
- **SM-2 spaced repetition** ‚Äî cards are scheduled based on your self-ratings (Again / Hard / Good / Easy)
- **Exam prep mode** ‚Äî curated Q&A ranked by importance with timed sessions and scoring
- **Activity heatmap** ‚Äî GitHub-style study streak tracker on the Stats page
- **Mastery tracking** ‚Äî per-deck mastery percentage with circular progress rings
- **Print / Save as PDF** ‚Äî dedicated print-optimised page for any deck
- **Backup export** ‚Äî download any deck as a formatted `.txt` study document
- **Dark mode** ‚Äî persisted via Zustand + localStorage
- **Google OAuth** ‚Äî per-user data isolation, every deck and session scoped to the signed-in user
- **Keyboard shortcuts** ‚Äî Space to flip, 1‚Äì4 to rate during study sessions

## Database Schema

```sql
-- Decks (scoped per user)
CREATE TABLE decks (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL DEFAULT 'local',
  title TEXT NOT NULL,
  description TEXT,
  source_filename TEXT,
  card_count INTEGER DEFAULT 0,
  mastery_percentage REAL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  last_studied_at DATETIME
);

-- Flashcards
CREATE TABLE cards (
  id TEXT PRIMARY KEY,
  deck_id TEXT NOT NULL,
  front TEXT NOT NULL,
  back TEXT NOT NULL,         -- JSON: { answer, explanation, mnemonic, keyFormula, examTip }
  card_type TEXT DEFAULT 'concept',
  repetitions INTEGER DEFAULT 0,
  easiness_factor REAL DEFAULT 2.5,
  interval_days INTEGER DEFAULT 0,
  next_review_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  last_reviewed_at DATETIME,
  FOREIGN KEY (deck_id) REFERENCES decks(id) ON DELETE CASCADE
);

-- Study activity (heatmap / streaks)
CREATE TABLE study_stats (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL DEFAULT 'local',
  date TEXT NOT NULL,
  cards_studied INTEGER DEFAULT 0,
  UNIQUE(user_id, date)
);

-- Exam sessions
CREATE TABLE exam_sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL DEFAULT 'local',
  deck_id TEXT,
  title TEXT NOT NULL,
  questions TEXT NOT NULL,   -- JSON array
  answers TEXT,
  score REAL,
  completed_at DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## Environment Variables

Create `.env.local` in the project root:

```env
# Anthropic Claude API
ANTHROPIC_API_KEY=sk-ant-...

# NextAuth
NEXTAUTH_SECRET=your-random-secret-here
NEXTAUTH_URL=http://localhost:3006

# Google OAuth (from Google Cloud Console)
GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret
```

### Getting credentials

**Anthropic API key:**
1. Go to [console.anthropic.com](https://console.anthropic.com)
2. API Keys ‚Üí Create new key

**Google OAuth:**
1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. APIs & Services ‚Üí Credentials ‚Üí Create OAuth 2.0 Client ID
3. Authorised redirect URIs: `http://localhost:3006/api/auth/callback/google`
4. For production add your deployed URL too

**NEXTAUTH_SECRET:**
```bash
openssl rand -base64 32
```

## API Routes

| Method | Route | Description |
|---|---|---|
| `POST` | `/api/generate` | PDF/PPTX text ‚Üí Claude ‚Üí flashcards + revision sheets |
| `GET` | `/api/decks` | List all decks for the signed-in user |
| `POST` | `/api/decks` | Create a new deck with cards |
| `GET` | `/api/decks/[id]` | Get deck with all cards and stats |
| `DELETE` | `/api/decks/[id]` | Delete deck and all its cards |
| `GET` | `/api/decks/[id]/export` | Download deck as formatted `.txt` |
| `GET` | `/api/study/[deckId]` | Cards due for review (SM-2 ordered) |
| `POST` | `/api/cards/[id]/review` | Submit SM-2 rating, reschedule card |
| `GET` | `/api/exam` | List exam sessions |
| `POST` | `/api/exam` | Generate a new exam from a deck |
| `GET` | `/api/exam/[id]` | Get exam with questions |
| `PATCH` | `/api/exam/[id]` | Submit exam answers, calculate score |
| `GET` | `/api/stats` | Study streak, daily history, heatmap data |

All routes return `401` if not authenticated, `403` if the resource belongs to another user.

## Spaced Repetition ‚Äî SM-2

Self-rating scale used during study sessions:

| Button | Quality | Meaning |
|---|---|---|
| Again | 0 | Completely forgot |
| Hard | 2 | Recalled with major difficulty |
| Good | 3 | Recalled after hesitation |
| Easy | 5 | Perfect recall |

A card is **mastered** when `repetitions >= 3` AND `easiness_factor >= 2.0`. New cards start at 0% mastery ‚Äî this is expected. Study consistently and mastery climbs automatically.

## Deployment

### Vercel

```bash
# Push to GitHub, then import at vercel.com
# Add all .env.local variables under Settings ‚Üí Environment Variables
# Update NEXTAUTH_URL to your production URL
# Update Google OAuth redirect URI to https://yourdomain.com/api/auth/callback/google
```

> **SQLite on Vercel:** Vercel serverless functions use `/tmp/` which is ephemeral. For persistent storage on Vercel, replace SQLite with [Turso](https://turso.tech) (SQLite-compatible, free tier). For a persistent filesystem, deploy on **Railway** or **Render** instead.

In `lib/db.ts` the DB path is already set to `/tmp/cramcraft.db` in production:

```typescript
const dbPath =
  process.env.NODE_ENV === 'production'
    ? '/tmp/cramcraft.db'
    : path.join(process.cwd(), 'cramcraft.db');
```

## Security

- API keys live only in `.env.local` ‚Äî never in client code or `NEXT_PUBLIC_` variables
- All `/api/*` routes call `getAuthUserId()` and return `401` if the session is missing
- Deck/exam ownership is verified on every write operation (`403` if user_id mismatch)
- `middleware.ts` protects all app routes ‚Äî unauthenticated users are redirected to `/auth/signin`
- Google OAuth uses `prompt: 'select_account'` to always show the account picker
